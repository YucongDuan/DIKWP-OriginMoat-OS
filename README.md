# DIKWP OriginMoat OS

**DIKWP OriginMoat OS** is a GitHub-ready open-source strategic release system for the DIKWP ecosystem. It helps an original creator publish useful open-source work while preserving attribution, official certification value, partner routes, training value, registry value, and commercial opportunity.

It is not a DRM system and does not prevent lawful open-source reuse. Instead, it turns every project into an auditable **Origin Capsule** and a strategic **Open-Core / Official-Service / Certification / Partner** map.

## Core idea

Code can be forked. Official origin, release chronology, signed registry status, certification authority, training programs, partner channels, and project-to-project ecosystem continuity are much harder to copy honestly.

## Features

- Portfolio-level DIKWP project analysis
- Open-core release layer planning
- Controlled official asset registry template
- License and attribution matrix
- Copycat risk scoring
- Partner program blueprint
- GitHub release sequence planning
- Official registry playbook
- DIKWP semantic and purpose-layer mapping
- Offline static boundary audit

## Install

```bash
pip install -e .
```

## Demo

```bash
originmoat analyze examples/sample_ecosystem.json --policy configs/default_policy.json --out outputs/demo
originmoat static-audit src --out outputs/demo/static_boundary_audit_report.json
```

## Important boundary

This tool is for open-source strategy, attribution integrity, official registry design, and partner ecosystem planning. It does not provide legal advice, trademark registration, copyright enforcement, or guaranteed commercial outcomes. Formal release, licensing, trademark, copyright, and enforcement actions should be reviewed by qualified professionals.
